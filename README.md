# 🍔 Mini Site Web – Burger King

Un mini‑projet front‑end reproduisant une page d'inspiration Burger King, avec une interface **moderne, responsive** et fidèle à l'univers visuel de la marque.

---

## 📌 Présentation

Ce projet est un mini site web statique réalisé dans le cadre de mon apprentissage personnel en développement web.

L'objectif était de recréer une page publicitaire Burger King en travaillant :

- ✅ La structure HTML sémantique
- ✅ La mise en page et le design en CSS
- ✅ L'intégration d'images et d'éléments graphiques
- ✅ La cohérence visuelle d'une marque existante
- ✅ Le **responsive design** (mobile, tablette, desktop)

---

## 🎨 Fonctionnalités

| Fonctionnalité | Description |
|---|---|
| 🏠 Page d'accueil | Bannière hero inspirée d'une campagne Burger King |
| 🍟 Actualités | Mise en avant des dernières offres et produits |
| 👑 Programme fidélité | Section dédiée avec visuel d'ambiance |
| 🍔 Sélection produits | Grille de produits avec effet au survol |
| 📱 Menu mobile | Hamburger menu fonctionnel (JavaScript) |
| 🔐 Dropdown login | Menu de connexion au survol de l'icône couronne |
| 📲 Responsive | Adapté mobile (≤ 680px), tablette et desktop |

---

## 🛠️ Technologies utilisées

- **HTML5** – Structure sémantique (`<section>`, `<article>`, `<footer>`, etc.)
- **CSS3** – Flexbox, variables CSS, media queries, animations
- **JavaScript** – Menu hamburger (toggle)
- **Font Awesome 6** – Icônes
- **Google Fonts** – Polices *Anton* (titres) et *Roboto* (texte)

## 📐 Responsive Design

Le site s'adapte à trois tailles d'écran :

| Breakpoint | Largeur | Comportement |
|---|---|---|
| Desktop | > 900px | Navigation complète, grilles larges |
| Tablette | ≤ 900px | Espacement réduit, tailles ajustées |
| Mobile | ≤ 680px | Menu hamburger, colonnes empilées |

---

## 🚀 Lancer le projet

Aucune installation requise — c'est du HTML/CSS/JS pur !

1. Clone ou télécharge le dossier
2. Ajoute tes images dans le dossier `/images/`
3. Ouvre `index.html` dans ton navigateur

---

## 📚 Ce que j'ai appris

- Utiliser les **variables CSS** (`--rouge`, `--jaune`, etc.) pour garder le code propre
- Créer un **menu responsive** avec JavaScript
- Maîtriser **Flexbox** pour les mises en page complexes
- Utiliser les **media queries** pour adapter le design à tous les écrans
- Écrire un HTML **sémantique** et accessible (`alt`, `aria-label`, etc.)
